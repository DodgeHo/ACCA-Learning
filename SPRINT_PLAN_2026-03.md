# AWS SAA Trainer 冲刺计划（2026-03）

## 1. 目标（本轮）
- 从“功能可用”推进到“可稳定长期使用”。
- 优先解决数据可靠性（尤其 AI 对话历史）与发布一致性问题。
- 在不大改架构前提下，补齐测试与文档，形成可重复发布流程。

## 2. 当前状态快照
- 跨平台 Flutter 刷题主流程已完成（筛选、错题循环、跳题、状态标记、AI 提问）。
- 本地存储已实现：原生 SQLite / Web IndexedDB。
- 主要风险：
  1) 原生 DB 修复流程会重建题库并仅回灌 `user_status`，`chat_history` 可能丢失。
  2) 文档与实现存在漂移（README 仍写 AI 接入 TODO，但代码已接 DeepSeek/OpenAI）。
  3) 自动化测试覆盖不足（当前仅 smoke test）。

---

## 3. 优先级与任务清单

### P0（本周必须完成）

#### P0-1 数据可靠性：修复 DB 重建导致聊天历史丢失
- 目标：当触发 `_needsNativeRepair` 时，除 `user_status` 外，完整保留 `chat_history`。
- 代码范围：`lib/db_native.dart`
- 工作内容：
  - 重建前导出 `user_status` 与 `chat_history`。
  - 重建后回灌两张运行时表。
  - 异常兜底：回灌失败时写日志并不中断题库加载。
- 预估：0.5 天
- 验收：
  - 构造 repair 场景后，历史标记与 AI 聊天都能保留。
  - 应用重启后历史仍存在。

#### P0-2 可恢复性：导出/导入聊天历史（MVP）
- 目标：提供“手动备份/恢复”能力，避免设备清理后不可恢复。
- 代码范围：`lib/main.dart`、`lib/app_model.dart`、`lib/db_native.dart`、`lib/db_web.dart`
- 工作内容：
  - 导出 JSON（按 `question_id -> content`）。
  - 导入 JSON（覆盖同题历史，校验格式）。
  - 设置页加入口与错误提示。
- 预估：1 天
- 验收：
  - 导出文件可在同端导入恢复。
  - 导入异常文件时提示明确，不崩溃。

#### P0-3 文档纠偏：README/INSTALL 与现状对齐
- 目标：避免误导后续维护与使用。
- 文档范围：`README.md`、`INSTALL.md`
- 工作内容：
  - 移除“AI 接入 TODO”表述，改为“已接入，支持 provider/model/baseUrl 配置”。
  - 增加“本地数据位置与备份建议”小节。
- 预估：0.5 天
- 验收：文档描述与 UI 实际一致，按文档可完成完整上手。

### P1（下周完成）

#### P1-1 测试补强（最小必要集）
- 目标：降低回归风险，尤其是状态与历史存储行为。
- 代码范围：`test/`（新增 3~5 个用例）
- 建议用例：
  1) `AppModel.mark` 在不同筛选模式下行为正确。
  2) `clearAllChatHistories` 后历史为空。
  3) `AiClient` 非 2xx 响应错误可读。
  4) 错题循环边界（首尾回环）逻辑。
- 预估：1~1.5 天
- 验收：`flutter test` 稳定通过，新增测试覆盖核心回归路径。

#### P1-2 Web 离线发布收口
- 目标：把“可离线回访”固化为可执行发布步骤。
- 范围：`web/sw.js`、`INSTALL.md`
- 工作内容：
  - 明确 `CACHE_VERSION` 更新规范。
  - 补充上线前后检查清单（在线首次加载、离线回访、数据持久化）。
- 预估：0.5 天
- 验收：按清单在 Chrome 可复现实测通过。
