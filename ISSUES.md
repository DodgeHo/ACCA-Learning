# Issues & Notes

## 已知限制

1. **主观题 OCR 质量波动**
   - 对扫描质量较差的 PDF，题干切分和答案抽取可能不稳定。

2. **图表题依赖额外映射**
   - 若 `assets/exhibits/index.json` 未配置，题面仍可用，但不会显示原题图表。

3. **Android 构建偶发 Kotlin 增量缓存告警**
   - 目前不影响最终 APK 产出；若持续失败建议 `flutter clean` 后重试。

4. **数据库 schema 受限策略**
   - 当前策略固定为不变更 schema，仅替换 `questions` 行，复杂结构信息需通过字段约定承载。

5. **本项目许可证**
   - 本仓库采用 MIT License（见 `LICENSE`）。

## 待改进

- 增加图表抽取与映射自动化脚本，减少手工维护 `index.json`
- 增加分科解析质量报告（分块成功率、缺失答案率、噪声率）
- 增加发布前自动验收脚本（产物存在性、哈希校验、切库冒烟）

## 贡献建议

- 提交规则修复时，请附带最小复现样本（原 PDF 片段或 OCR 块）
- 新增脚本优先保持确定性与可复跑，不直接手改数据库
